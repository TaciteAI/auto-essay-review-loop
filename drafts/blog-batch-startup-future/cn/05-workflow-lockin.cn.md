# 工作流锁定是新的切换成本

2026 年最强的 AI 切换成本，不是一条合同条款，而是一个已经在公司 CRM、客服队列、日历、计费系统和内部规则里运行了 18 个月的 agent。当软件开始采取行动，而不只是生成文字，替换供应商就不再是采购决策，而接近一次运营手术。

这就是为什么 system of action 比旧的 system of record 更关键。Salesforce、ServiceNow、HubSpot、Epic、NetSuite、Workday 仍然掌握关键数据。但战略价值正在向动作层移动：读取数据、判断下一步、执行、记录结果、从异常里学习的 agent。买方要的是行动，不是文本生成。

资本也在追这个位置。2026 年 agentic AI 平均融资额接近翻倍到 [$155M](https://agentmarketcap.ai/blog/2026/04/08/agentic-ai-funding-velocity-2026-sector-map-vertical-distribution)，2025 年 AI 拿走全球 VC 投资的 [61%](https://agentmarketcap.ai/blog/2026/04/10/ai-captures-50-percent-global-venture-capital-2025)。钱追的不是抽象智能，而是工作流里的控制点。

## Agent 会学会这家公司

传统 SaaS 的锁定来自数据存储、用户培训、系统集成和采购疲劳。AI agent 多了一层更贴身的东西：运营记忆。一个处理客服工单的 agent，会学会哪些客户必须升级、哪些退款可以批准、哪些话术会触发法务、哪些客户经理更能承担风险。一个跑销售运营的 agent，会学会区域划分、折扣习惯、买方反对意见、续约时点和 CRM 卫生标准。

这些知识不会完整写在数据库 schema 里。它们存在于修正、例外、偏好和反复决策中。agent 之所以有用，是因为它吸收了本地工作的纹理。移除它，不只是换供应商，而是换掉一个已经训练过一部分的操作员。

这也是工作流集成构成护城河的原因。NFX 对 [AI defensibility](https://www.nfx.com/post/ai-defensibility) 的分析强调，防御性来自采用系统和网络效应，而不是孤立功能。在 agentic software 里，采用不是登录一次，而是反复获得在业务系统里行动的许可。

## 切换会制造隐藏风险

企业买方害怕切换，不是因为新工具难学，而是怕业务断裂。如果一个 agent 一直在更新 CRM 阶段、发送 follow-up、路由工单、准备发票、标记合规例外，替代品必须复现大量小行为，才有资格被信任。

表面的迁移清单会漏掉最难部分。数据可以导出，集成可以重建。但新 agent 必须重新学习隐性工作流规则、升级阈值、边界案例历史、组织里的政治地图。成熟买家正在把每类工作流里的 agent 供应商减少到一两个，这会让已经嵌入的既有玩家获得强优势。

中国市场的例子会更明显。一个在企业微信、钉钉、飞书、纷享销客、销售易或自研 CRM 里跑了 18 个月的 agent，会积累大量本地流程知识。后来者即使用了更强模型，也要重新走权限、数据隔离、审批链、IT 安全和业务试点。benchmark 高一点，不一定能抵消这些成本。

## 护城河来自把活干好

最强的工作流锁定来自可靠执行，而不是黑暗模式。如果 agent 省钱、缩短周期、抓住错误，并且随使用变好，客户留下它是因为替换不理性。如果它只是把数据锁在糟糕的导出流程后面，客户迟早会惩罚供应商。

这正是 service-as-software 和 workflow lock-in 汇合的地方。Sequoia 的 thesis 是 AI 公司应该卖结果，而不是卖 seat。交付结果要求供应商贴近工作。越贴近工作，学到的越多；学到的越多，越难被替换。

对创始人来说，结论很具体：不要做一个等在另一个标签页里的工具。要做一个能在钱、风险和客户承诺所在系统里赢得权限的 operator。下一批伟大的 AI 公司，不会因为用户和它聊天而被记住，而会因为把它拔掉会让业务停摆而被留下。
