# 多人 Agent 网络效应

AI 的网络效应容易被夸大，也容易被低估。一个消费级聊天机器人不会因为用户多就天然更强，某个用户的聊天记录也很难以专有方式改善另一个用户的体验。但在垂直行业里，Agent 的网络效应更隐蔽也更真实：共享的数据结构、共享的对手方行为、共享的异常处理、共享的行业协议。

过去的网络效应是人连接人。AI 时代的许多网络效应，是系统连接系统。它不像社交网络那样显眼，却可能在跨机构工作流里更值钱。

## 单人 Agent 有天花板

第一代企业 Agent 多数是单人工具：总结会议、起草邮件、分类工单、生成病历、搜索知识库。它们有用，但边界清楚。Agent 活在一家公司内部，优化一个人的任务。

更大的机会出现在天然多人协作的流程里。保险理赔涉及保险公司、经纪、定损、维修厂、医院和律师。物流涉及货主、承运商、仓库、海关和支付。医疗涉及医院、医保、药房、检验机构和患者。法律交易涉及甲方、乙方、外部律师和法院。

Bessemer 对垂直 AI 的判断是，[行业 AI 可以拥有深工作流](https://www.bvp.com/atlas/vertical-ai)。下一步不是更会聊天，而是在同一行业内连接更多参与者，让 Agent 成为机构之间的翻译层和执行层。

## 网络是操作记忆

AI 的网络效应不一定来自更多训练数据，也可以来自操作记忆。一个理赔 Agent 如果看见不同保险公司反复出现的拒赔原因，就能更快提示补充材料。一个法务 Agent 如果知道某类条款在哪些交易中容易被接受，就能更快生成谈判路径。一个供应链 Agent 如果知道哪些港口、仓库、承运商经常延迟，就能提前调整动作。

这并不要求泄露客户数据。产品需要学习的是归一化模式：哪些异常常见、哪些材料能解决、哪些格式被接受、哪些升级路径有效、哪些规则发生了变化。中国的字节、阿里、腾讯云、百度智能云都可以提供底层能力，但行业操作记忆需要在真实流程里长期积累。

这种记忆会复利。新客户加入时，买到的不只是软件，而是一张已经被前人付费探索过的行业地图。

## Agent 之间需要协议

流行讨论常问 Agent 是浏览网页还是调用 API。垂直网络里更关键的问题是，Agent 能否在权限清楚的前提下与另一个 Agent 协商结构化任务。理赔 Agent 向维修 Agent 请求证据，采购 Agent 向供应商 Agent 请求合规文件，法务 Agent 与对方系统交换修订立场，这些都不需要科幻式自治。

它们需要身份、权限、审计日志、任务状态和结构化消息。Salesforce 发布[Agentforce](https://www.salesforce.com/news/stories/agentforce-ai-agents-announcement/)说明，巨头也意识到 Agent 会靠近业务流程。但垂直创业公司能在更窄的跨机构场景里做得更深。

真正的网络效应会在每个参与者单独安装时就有价值，并在相邻参与者加入后变得更好。这不是病毒传播，而是工作流图谱的复利。未来的强公司不会只因为模型更聪明而赢，而会因为每个客户都让共享操作记忆更完整。
