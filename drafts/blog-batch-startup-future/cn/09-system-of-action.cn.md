# 行动系统

企业软件过去围绕记录系统展开。Salesforce 拥有客户记录，Workday 拥有员工记录，ServiceNow 拥有工单，Epic 拥有病历。记录是事实来源，拥有记录就能获得稳定经济价值。

AI 正在把价值线推向行动系统。记录仍然重要，但不再足够。一个记住客户的数据库，价值不如一个能判断下一步、起草回复、更新字段、触发流程、留下审计链的 Agent。未来的企业界面不是更漂亮的表格，而是被授权的执行。

## 记录必要但被动

传统 SaaS 的承诺是集中数据、标准化流程、报告活动。这创造了 Salesforce、Workday、ServiceNow 这样的巨头，因为分散记录曾经非常昂贵。但二十年后，许多企业已经有记录系统。新的痛点不是存事实，而是在事实之间完成工作。

销售团队不想再填一个 CRM 字段，而是希望跟进邮件发出去、商机阶段更新、续约风险提醒、经理同步完成。法务团队不想再多一个文档库，而是希望条款被比较、风险被分类、谈判路径被准备。财务团队不想再看仪表盘，而是希望异常对账被处理。

Salesforce 发布[Agentforce](https://www.salesforce.com/news/stories/agentforce-ai-agents-announcement/)的重要性就在这里：记录系统必须变成行动系统，或者被行动系统包围。

## 行动层拥有用户

行动层有三个优势。第一，它位于用户意图出现的地方。第二，它在执行过程中产生新的结构化数据。第三，它会让底层记录系统逐渐基础设施化。

如果一个 Agent 规划客户电话、写跟进邮件、更新 CRM、创建任务、提醒经理，用户的忠诚会转向 Agent。Salesforce 仍然重要，但更像基础设施。同样的压力会影响 Workday、金蝶、用友、钉钉、企业微信以及各类行业系统。

这不意味着创业公司必须替换巨头。替换成本高，政治阻力大。更好的入口是拥有一个高频、痛苦、记录系统做得不好的动作，然后向相邻动作扩张。

## 执行会产生更好的数据

记录系统里的数据常常滞后、不完整，甚至带有表演性质，因为人是在工作之后补录信息。Agent 可以反过来：先执行或协调工作，记录作为执行副产物自动生成。

销售 Agent 如果听电话、写邮件、更新阶段、追踪异议，就比周五下午手填的 CRM 更接近真实业务。理赔 Agent 如果收集证据、处理异常、记录拒赔原因，就比静态案卷更有过程价值。法务 Agent 如果记录某个条款为什么接受或升级，就创造了文档库从未拥有的组织记忆。

Gartner 预测 2025 年全球 IT 支出将达到[$5.74 trillion](https://www.gartner.com/en/newsroom/press-releases/2024-10-23-gartner-forecasts-worldwide-it-spending-to-grow-nine-point-three-percent-in-2025)。但预算不会长期奖励 AI 表演，它会奖励减少人力、压缩周期、让下一步可衡量的系统。

## 记录层会反击

巨头不会旁观。它们有分发、权限、采购路径和数据重力。阿里、腾讯、百度也会把 Agent 放进云、办公和行业软件。但广义平台必须泛化，而最深的行动流程往往具体、受监管、例外多、组织政治重。

创业公司的机会，是把记录系统当作地形，而不是正面战场。谁发起行动，谁占据注意力；谁记录行动理由，谁积累组织记忆；谁让下一步发生，谁拿走利润。记录系统回答发生了什么，行动系统回答接下来该做什么。AI 时代，利润会向第二个问题移动。
